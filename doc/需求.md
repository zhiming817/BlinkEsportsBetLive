# 📝 产品需求文档 (PRD): BlinkBet Live

**版本：** v1.0 (Hackathon Edition)
**状态：** 草案
**目标：** 在 2026 Solana 黑客松中展示 Web3 社交预测的极致体验。

---

## 1. 产品概述
**BlinkBet Live** 是一个基于 Solana Actions & Blinks 的垂直电竞预测协议。它允许用户无需离开社交媒体（Twitter/X、Discord、Telegram），通过点击一个简单的交互卡片，即可对正在进行的电竞比赛（如 LoL, Dota 2, CS2）中的“微型事件”进行毫秒级下注与结算。

### 核心价值主张
* **无感交互：** 告别连接 dApp、切换窗口的繁琐流程。
* **即时快感：** 针对“下一次团战”、“下一血”等短周期事件，结果即出，资金即回。
* **极致成本：** 利用 ZK Compression 降低高频下注的存储开销。

---

## 2. 核心功能需求

### 2.1 动态 Blink 预测卡片 (Action/Blink)
* **实时状态渲染：** Blink 卡片需显示当前比赛对阵、预测题目（如：哪方先拿小龙？）、当前赔率。
* **多选项支持：** 支持 2-4 个快速点击按钮（如：Option A, Option B）。
* **支付集成：** 一键调用 Solana 钱包插件（Phantom/Backpack）签名，支持 $SOL$ 和主流 $USDC$。

### 2.2 实时预测引擎 (Program Layer)
* **Parimutuel 池模型：** 采用等额彩票制算法，根据两侧资金比例动态调整赔率，确保协议无对手方风险。
* **毫秒级锁仓：** 当后端检测到关键事件发生（如团战开始）时，通过 **Action Chaining** 立即置灰 Blink 按钮，防止套利。

### 2.3 自动化结算系统 (Oracle & Agent)
* **数据喂价：** 集成 2026 年主流的拉取式 Oracle（如 Pyth），通过 API 获取游戏内原始数据。
* **AI 裁判：** 使用 AI Agent 监听游戏 API 结果，一旦数据达成（如“First Blood”达成），立即触发链上结算合约，将奖金自动分发至用户钱包。

## 3. 技术架构 (Technical Stack)

| 模块 | 技术选型 | 备注 |
| :--- | :--- | :--- |
| **智能合约** | Anchor / Rust | 部署在 Solana Mainnet (Firedancer 客户端优化版) |
| **状态存储** | **ZK Compression** | 用于存储海量用户的历史预测数据，降低租金成本 |
| **前端协议** | Solana Actions & Blinks | 符合最新的 Dialect 交互标准 |
| **后端/API** | Node.js (TypeScript) + Redis | 处理实时赔率计算与高并发 Action 请求 |
| **数据源** | PandaScore API + Pyth | 获取电竞实时数据与代币价格 |

## 4. 业务流程 (User Flow)

1.  **分发：** 官方或 KOL 在 X 上发布带有 Action URL 的推文。
2.  **解析：** Twitter 插件将 URL 解析为 **Blink 交互卡片**。
3.  **预测：** 用户点击“左队先拿一塔”，钱包弹出金额确认。
4.  **上链：** 交易通过 Solana 网络执行，用户获得对应的预测凭证（基于 Token Extensions 的临时状态）。
5.  **结算：** 比赛事件发生后 3-5 秒内，AI Agent 确认结果，合约自动将 $SOL$ 奖励推送到中奖用户钱包。

---

## 5. 关键指标 (KPIs) - 黑客松评审重点

* **TPS (每秒预测数)：** 展示在高频电竞环境下，合约处理并发请求的能力。
* **End-to-End Latency：** 从点击按钮到交易确认的延迟（目标：<$500ms$）。
* **存储成本：** 使用 ZK Compression 后，单次下注对比传统账户模型节省的租金百分比。

## 6. 路线图 (Milestones)

* **M1 (Day 1-2):** 完成 Anchor 合约核心逻辑，实现 Parimutuel 资金池管理。

* **M3 (Day 4-5):** 对接电竞实时数据源，完成自动结算闭环。
* **M4 (Day 6):** 优化 Blink 动态封面（根据比赛进度实时生成的图片）。
* **M5 (Day 7):** 录制演示视频，重点展示从“看推特”到“拿奖金”的全流程。
